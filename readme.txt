Небольшое описание:


Для выполнения задания использовал БД Apache Derby, которая включается в JDK.
Синтаксис некоторых запросов отличается от ораклового, кроме того немного
отличаются типы данных, но смысл надеюсь не поменялся )

Установок БД никаких не требуется, самая верхняя панель исключительно для 
проверки коннекта к БД и наличия драйвера, можно приступать сразу к тестированию.

Архив включает в себя JRE javа 8, исключительно для удобства,и уже созданную БД 
на 300 заявок,так что можно ничего не генерировать и сразу выполнять тесты, переход к сл.
или пред. тесту осуществляется по кнопкам над панелью тестирования.

Для генерации скрипта создания БД можно указать кол-во заявок которое необходимо 
внести в БД.После нажатия кнопки Сгенерировать программка по шаблону создаст скрипт 
создания БД, После этого можно выполнить создание БД (создать).

К сожалению не успел вынести в скрипты шаблоны вставки записей в таблицы.
Так что сейчас они захардкожены и используют синтаксис Derby.
Все запросы содержаться в папке sql, исходники в src, в drb — Apache Derby

createTemplDb.sql  - скрипт шапка для создания скрипта создания БД
createDb.sql - Скрипт для создания БД c вставкой указнанного кол-ва заявок.

add.sql - скрипт создания дубликатов заявок
check.sql - проверка наличия дубликатов заявок
del1.sql del2.sql  - скрипты удаления дубликатов, для сравнения производительности
t1.sql t2.sql t3.sql t4.sql - скрипты для тестов.

Для запуска st_jre.cmd — с использование включенного в архив JRE и st.cmd  — 
с установленной на компьютере java ( должна быть 8+)
